# Spec: Health Check Endpoints

**Version:** 0.1.0
**Created:** 2026-04-05
**PRD Reference:** docs/prd.md
**Status:** Complete

## 1. Overview

Add health check endpoints to both the frontend and BFF that return app status, version, and build metadata. The frontend serves a static JSON file generated at build time. The BFF returns detailed build info including git commit, branch, Node.js version, OS, architecture, and build timestamp.

### User Story

As an operator monitoring Rx Drill, I want health check endpoints that report version and build metadata so that I can verify which build is deployed and confirm services are healthy.

## 2. Acceptance Criteria

| ID | Criterion | Priority |
|----|-----------|----------|
| AC-001 | `GET /health` on the frontend returns JSON with `status` and `version` fields | Must |
| AC-002 | The frontend `version` is read from `package.json` at build time | Must |
| AC-003 | The frontend `status` is always `"healthy"` (static file, if nginx serves it the app is up) | Must |
| AC-004 | The frontend `/health` response is a static JSON file generated at build time (`public/health.json`) | Must |
| AC-005 | `GET /api/health` on the BFF returns JSON with `status`, `version`, `git_commit`, `git_branch`, `node_version`, `os`, `arch`, and `build_time` | Must |
| AC-006 | BFF `version` is read from `bff/package.json` at build time | Must |
| AC-007 | BFF `git_commit` is the short SHA of the commit the image was built from | Must |
| AC-008 | BFF `git_branch` is the branch the image was built from | Must |
| AC-009 | BFF `node_version` is the Node.js runtime version (e.g., `v22.x.x`) | Must |
| AC-010 | BFF `os` is the target operating system (e.g., `linux`) | Must |
| AC-011 | BFF `arch` is the target architecture (e.g., `x64`) | Must |
| AC-012 | BFF `build_time` is a UTC ISO 8601 timestamp of when the image was built | Must |
| AC-013 | BFF build metadata is embedded at Docker build time via build args | Must |
| AC-014 | Both endpoints return `Content-Type: application/json` | Must |
| AC-015 | Both endpoints are unauthenticated (no JWT required) | Must |
| AC-016 | The existing `GET /api/health` response format changes from `{"status":"ok"}` to the full build info response | Must |

## 4. Data Model

### Frontend Build Output (`public/health.json`)

| Field | Type | Source |
|-------|------|--------|
| status | string | Always `"healthy"` |
| version | string | `package.json` version |

Generated by `scripts/generate-seo.ts` (or a new script) at build time. Gitignored.

### BFF Build Info (`bff/src/build-info.ts`, generated)

| Field | Type | Source |
|-------|------|--------|
| version | string | `bff/package.json` version |
| git_commit | string | `GIT_COMMIT` build arg (short SHA) |
| git_branch | string | `GIT_BRANCH` build arg |
| node_version | string | `process.version` at runtime |
| os | string | `process.platform` at runtime |
| arch | string | `process.arch` at runtime |
| build_time | string | `BUILD_TIME` build arg (ISO 8601 UTC) |

## 5. API Contract

### GET /health (frontend, served by nginx)

**Auth:** None
**Response (200):**
```json
{
  "status": "healthy",
  "version": "0.5.8"
}
```

### GET /api/health (BFF)

**Auth:** None
**Response (200):**
```json
{
  "status": "healthy",
  "version": "0.5.8",
  "git_commit": "554bddd",
  "git_branch": "main",
  "node_version": "v22.14.0",
  "os": "linux",
  "arch": "x64",
  "build_time": "2026-04-04T18:13:00Z"
}
```

## 6. Implementation Notes

### Frontend

- `scripts/generate-seo.ts` (or new script) writes `public/health.json` at build time
- Reads version from `package.json`
- Gitignore `public/health.json`
- nginx serves it as a static file at `/health`

### BFF

- `bff/Dockerfile` passes build args: `GIT_COMMIT`, `GIT_BRANCH`, `BUILD_TIME`
- A generated file `bff/src/build-info.ts` embeds the static values
- `bff/src/index.ts` health endpoint reads from build-info + `process.version` / `process.platform` / `process.arch`
- CI workflows pass the build args during `docker build`

### nginx

- Add `location = /health` that serves `public/health.json` directly (before the SPA fallback)

## 7. Edge Cases

| Case | Expected Behavior |
|------|-------------------|
| Build without git args (local dev) | BFF returns `"dev"` for git_commit/branch, current timestamp for build_time |
| Frontend health.json missing | nginx returns 404 (indicates broken build) |
| BFF is down but nginx is up | `/health` returns 200, `/api/health` returns 502 |

## 8. Dependencies

- `package.json` version field (already maintained)
- Docker build args in CI workflows
- nginx config (add `/health` location)
- `scripts/generate-seo.ts` (extend or new script)
